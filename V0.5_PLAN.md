# CodeGate v0.5 版本方案

**规划日期**: 2026-05-02
**目标版本**: Alpha v0.5.0
**预计周期**: 4 天

---

## 一、v0.5 总主题：Evidence Quality & Real Executor Evaluation

一句话目标：CodeGate 不只要抓住 Silent Behavioral Drift，还要能在 artifact 和 demo 里清楚证明：**抓到了什么、来自哪里、为什么这个决策合理。**

---

## 二、关键里程碑

### Milestone 1: Evidence Explainability (P0) - [DONE]
**目标**: Representative security rule triggers in SEC-1 ~ SEC-10 emit structured `evidence`.

**完成项**:
- [x] 实现 `_build_evidence` 助手，包含 `file / line / pattern / snippet`。
- [x] 覆盖 SEC-1 ~ SEC-10 代表性规则触发场景。
- [x] 增加 `severity` 和 `reason` 字段解释决策。
- [x] 保留旧字段（`removed`, `added`）确保向后兼容。
- [x] 增加 `tests/test_security_evidence.py` 自动化验证。

### Milestone 2: Safe Refactoring Suite (P1) - [DONE]
**目标**: 验证常见的提取器可见重构（T13-T16）保持 non-blocking (advisory)。

**完成项**:
- [x] 新增 Python/FastAPI 等价重构 fixtures (T13-T16)。
- [x] 更新 `run_demo.py` 分组展示 "Blocking drift" 与 "Safe refactor"。
- [x] 优化提取器正则，支持常用重构命名（如 `get_org_context`, `require_admin_role`）。
- [x] 验证 T13-T16 均不产生 revise_code 或 escalate_to_human。

### Milestone 3: Public CI Demo (P1) - [DONE]
**目标**: 外部用户 fork 后能通过 GitHub Actions 看到复现结果。

**完成项**:
- [x] 增加 `.github/workflows/ci.yml`。
- [x] CI 自动运行 `pytest` 和零 LLM demo。
- [x] README 增加 CI 状态徽章。

### Milestone 4: Real Executor Evaluation (P3) - [OUT OF SCOPE FOR v0.5.0]
**目标**: 真实执行器（Codex, Gemini）在安全漂移场景下的表现评估。
